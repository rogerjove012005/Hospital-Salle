Proyecto académico

laSalle Health Center

Sistema inteligente de soporte hospitalario — IA, Big Data y automatización.

Presentación 10–15 min · Docker · FastAPI · PySpark · Radiología IA

1. Problema y objetivo

  • Digitalizar ingesta de datos clínicos heterogéneos (CSV).
  • Apoyar decisiones con analítica y clasificación de radiografías RX.
  • Portal unificado para paciente, médico y administración.

2. Arquitectura

  • PostgreSQL — datos estructurados y eventos.
  • MinIO — archivos CSV archivados.
  • FastAPI — REST, JWT, RBAC.
  • Workers — ingesta CSV + PySpark.
  • Frontend nginx — portal estático.

3. Pipeline de datos

mock-hospital-feed → csv-ingest-worker → POST /imports/csv → calidad → spark-csv-aggregate → dashboard.

50k+ filas agregadas en demo · Parquet en volumen Docker.

4. Centro de control

  • KPIs operativos y gráficos Chart.js.
  • Alertas (pipeline + data_quality_issues).
  • Informe HTML automatizado.

5. Radiología IA

  • Clases: Sana, Neumonía, COVID-19.
  • Métricas, matriz de confusión, inferencia POST /radiology/predict.
  • Disclaimer ético: no es diagnóstico médico.

6. Seguridad y ética

  • JWT, roles, minimización de datos por perfil.
  • RGPD — datos sintéticos / demo.
  • Documentación en docs/ethics/.

7. Demo en vivo

  1. Login admin → landing.
  2. Centro de control + informe.
  3. Radiología: métricas + predict.
  4. Ingesta CSV (imports).

http://localhost:3000

8. Conclusiones

  • Stack containerizado reproducible con Compose.
  • Trazabilidad: eventos, calidad, observabilidad /health/observability.
  • Trabajo futuro: tests E2E, cluster Spark, despliegue cloud.
